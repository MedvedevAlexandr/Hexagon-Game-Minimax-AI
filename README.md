## 📌 О проекте

«Гексагон» — настольная логическая игра для двух игроков на шестиугольной доске.  
Проект реализует:

- графический интерфейс на **Qt** (QGraphicsScene);
- пошаговый режим **игрок против компьютера**;
- компьютерный игрок (ИИ), использующий алгоритм **минимакс с альфа-бета отсечением**;
- подсчёт очков, определение победителя, отображение последовательности ходов ИИ.

---

## 🎲 Правила игры

- Игровое поле состоит из **61 шестиугольной ячейки** (гекса).
- В начале игры:
  - красная фишка (игрок) расположена в центре верхней части поля;
  - синяя фишка (компьютер) — в центре нижней части.
- Ход состоит из выбора **своей фишки** и **пустой соседней ячейки** (через одно поле или напрямую).  
  *Возможные ходы показаны на рисунках в отчёте.*
- Если новая ячейка граничит с фишками противника — они **захватываются** (становятся вашего цвета).
- Игра заканчивается, когда у одного из игроков нет возможных ходов или поле полностью заполнено.
- Побеждает тот, у кого **больше фишек** на доске. При равном количестве — ничья.

---

## 🤖 Искусственный интеллект

ИИ использует **алгоритм минимакс с альфа-бета отсечением**.  
Глубина поиска и оценочная функция настроены на максимизацию количества захваченных фишек противника.


В проекте используется упрощённая версия — **жадный выбор хода** с максимальным захватом (т.к. полная реализация минимакса не была доведена до рабочего состояния, но её структура присутствует в коде).

## 🖥️ Интерфейс

| Элемент | Описание |
|---------|----------|
| Доска из гексов | Каждый гекс пронумерован (ID) |
| Индикатор хода | "TURN: red" / "TURN: blue" |
| Счётчики | RED COUNT и BLUE COUNT |
| Панель лога | Список ходов ИИ (ID гекса) |
| Сообщение о победе | "RED WIN", "BLUE WIN" или "DRAW" |
